Resumo das Alterações

- Normalização do lblScanner
  
  - Remove prefixo e sufixo “..:::” e “:::..” do texto.
  - Anexa lblPromoPdv ao lblScanner em nova linha, evitando duplicação.
  - Implementação: função normalize_scanner_text e uso no parsing.
  - Referências:
    - main.py: extract_element_info + normalize_scanner_text
    - main.py: parsing dos elementos (inclui lblScanner e concatenação com lblPromoPdv)
- Quebra de Linha na Interface
  
  - Consulta: elementos adicionais respeitam “\n”.
  - Admin: “Código do Produto” respeita “\n”.
  - Referências:
    - consulta.html (whitespace-pre-line)
    - admin.html (whitespace-pre-line)
- Endpoint de Limpeza de Cache
  
  - Garantido funcionamento de POST /api/limpar-cache.
  - Ajuste estrutural: app.run movido para o final do arquivo para todas as rotas estarem carregadas.
  - Testado o fluxo de limpeza total e por rota.
  - Referência:
    - main.py: /api/limpar-cache
    - main.py: bloco do app.run ao final
- Serviço Windows (iniciar com o Windows)
  
  - Instalador do serviço atualizado para preferir o executável da aplicação, caso exista, ou rodar via Python.
  - Variável de ambiente do serviço definida: RUNNING_AS_SERVICE=1 (desativa tray).
  - Logs com rotação configurada.
  - Referência: instalar_servico.bat
- Criação do Executável de Inicialização
  
  - Executável gerado: API_Unidasul_Service.exe
  - Geração automatizada via criar_executavel_servico.py com modo CI (sem prompts).
  - Script corrige problemas de interação e ícone/elevação.
  - Batch de criação fortalecido: criar_executavel.bat
    - Detecta Python/py, valida pip, instala PyInstaller, valida arquivos essenciais.
- Manual Atualizado
  
  - Guia de criação/uso e instalação como serviço (services.msc, firewall, logs).
  - Referência: README_EXECUTAVEL.md
Impacto no Comportamento da API

- Respostas do endpoint de consulta
  - lblScanner agora vem limpo (“..::: … :::..” removidos).
  - lblScanner inclui lblPromoPdv em linha separada quando existir.
  - lblPromoPdv permanece disponível em elementos (mantido para compatibilidade).
- Limpeza de cache
  - POST /api/limpar-cache funcional (JWT necessário). Pode limpar tudo ou por rota com parâmetro rota=dashboard_data, get_logs, etc.
Se quiser, posso remover o lblPromoPdv do payload e deixá-lo apenas dentro do lblScanner, ou aplicar a exibição com quebra de linha em outras telas/relatórios.
O que ajustei para viabilizar a criação automatizada:

- Corrigi o gerador para suportar modo não interativo e evitar travar em prompts.
- Gerei o EXE via PyInstaller com ícone e elevação configurados.
Manual para instalação no Windows Server

- Atualizei o guia com passo a passo consolidado em README_EXECUTAVEL.md:
  - Copiar pasta do projeto + API_Unidasul_Service.exe + instalar_servico.bat/desinstalar_servico.bat
  - Executar instalar_servico.bat como Administrador
  - Verificar o serviço APIUnidasul em services.msc
  - Acessar http://localhost:5000/
  - Regra de firewall opcional para a porta 5000
  - Caminho dos logs
